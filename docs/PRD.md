# 慧对接 SmartConnect — PRD 产品需求文档

> 版本: v1.0  ·  日期: 2026-07-01  ·  状态: 草案

---

## 1. 产品概述

### 1.1 产品定位
慧对接是面向项目方与项目合作者的精准匹配平台,核心口号:**"向割韭菜说不,为您精准匹配好项目"**。

平台通过标准化项目展示、三维评分体系、专家分析,降低项目甄别成本,连接项目方(B端)与合作者(C端)。

### 1.2 三端结构

| 端 | 使用者 | 入口 | 核心职责 |
|---|---|---|---|
| 小程序 | C端用户 | 微信小程序 | 浏览项目/合作/加入/推荐 |
| B端后台 | 项目方(=城市运营中心) | PC Web | 管理项目/用户/财务 |
| 总后台 | 慧对接平台 | PC Web | 管理B端/数据/财务/模块 |

### 1.3 多租户(千人千面)
- 同一小程序,按 URL/scene 参数携带 `b_id`(B端ID)渲染专属内容
- 每个B端获独立小程序码及链接
- C端入口二分:
  - **慧对接主入口**: 全功能(首页/项目/合作/我的完整)
  - **纯B端入口**: 仅2功能(项目介绍 + 我的[认证+下级用户])

## 3. 小程序(C端)需求

### 3.1 首页

| 模块 | 说明 |
|---|---|
| 数据显示框 | 显示用户数/最近浏览/正在浏览项目的用户(顶部) |
| 广告语模块 | "向割韭菜说不,为您精准匹配好项目"(数据框下) |
| 合作动态 | 项目合作动态资讯(主模块) |

> B端专属首页: 显示该B端项目浏览人数、最近浏览用户、项目合作/发展动态

### 3.2 项目

#### 3.2.1 项目类别
顶部展示类别标签: AI、直播、电商等(平台维护类别字典)

#### 3.2.2 搜索
按地区、行业等关键词搜索项目

#### 3.2.3 项目展示(标准模板)

每个项目按标准模板展示:

| 字段块 | 内容 |
|---|---|
| 一句话介绍 | 项目核心简介 |
| 项目简介 | 详细描述 |
| 项目愿景 | 长期愿景 |
| 项目发展及目标 | 阶段目标 |
| 项目团队 | 团队成员介绍 |
| 项目技术 | 技术栈/方案 |
| 市场竞争 | 竞品分析 |
| 项目需求 | 资源/合作需求 |
| SWOT分析 | 优势/劣势/机会/威胁 |
| 公司简介 | 成立时间、注册资金、经营范围 |
| 企业风险查询 | 企查查链接(手动填) |
| 创始人介绍 | 创始人背景 |
| 项目发心 | 创立初心 |
| 商业计划书BP | **仅BP会员可读**(按项目收费解锁) |

#### 3.2.4 项目评价(三维评分)

| 维度 | 颜色 | 初始来源 |
|---|---|---|
| 真实性 | 绿色 | 慧对接平台打分 |
| 风险性 | 红色 | 慧对接平台打分 |
| 盈利性 | 金色 | 慧对接平台打分 |

- 表示形式: 三条颜色框百分比,每框5小格,每格20分
- 或 高/中/低
- 首次由平台打分,后续由用户五星评价动态生成

#### 3.2.5 专家分析
平台产出的项目专业分析(✅标记)

#### 3.2.6 合作要求
C端加入该项目的门槛要求描述

#### 3.2.7 发布项目
- 新用户申请发布项目: 填写项目信息 + 支付发布费 **[v0.2 接入支付; v0.1 人工审核免费发布]**
- 审核模式: 系统默认通过 或 人工审核(可配置)
- 审核通过后,系统向申请手机号发送**一次性初始化登录链接**(短信,带签名token,有效期30分钟)
- B端首次登录通过该链接设置密码,不使用静态默认密码
- 老用户可在PC后台或此处发布/编辑项目

> ⚠️ 旧设计"默认密码 `huiduijie`"已废弃,存在批量撞库与首登未改密风险。

| 我收藏的项目 | 收藏列表 |
| 我加入的项目 | 已加入项目列表 |
| 我的下级用户 | 分享二维码/链接获取的下级(一级) |

## 6. 评分体系详细

### 6.1 三维评分
- **真实性**(绿): 项目真实程度
- **风险性**(红): 投资风险等级
- **盈利性**(金): 盈利能力

### 6.2 评分规则

1. 项目首次上线: 慧对接平台打初始分
2. 用户加入项目后可评价(五星)
3. 最终分公式:
   `最终分 = 平台初始分 × W_init + 用户均分 × W_user` (归一化到0-100)
   - 默认权重: `W_init = 0.4`, `W_user = 0.6`
   - **最小评价数保护**: 用户评价数 < 3 时, `W_user = 0`(仅显示平台分), 避免小样本失真
4. 展示: 5格进度条(每格20分) 或 高/中/低

### 6.3 分数方向(关键)
- **真实性**(绿): 分**越高越好**(100=完全真实)
- **盈利性**(金): 分**越高越好**(100=盈利能力强)
- **风险性**(红): 分**越高代表风险越大**(100=极高风险, 0=无风险)
  > 展示时风险性进度条满格=高风险,需配套警示文案。颜色框仍为红色但语义为"风险等级"非"安全等级"。

### 6.4 刷分防控
- 仅已加入项目且加入时长 > 24h 的用户可评价
- 同一用户对同一项目仅一条评价,可修改(修改触发重算)
- 异常评价过滤: 同IP/同设备短时批量评价→标记可疑,不纳入聚合
- 平台可对单项目评分重置/锁定(风控)

### 6.5 评价权限
- 仅已加入项目用户可评价
- 每用户每项目评价一次,可修改

## 7. 会员与付费

> 版本归属: v0.1 仅免费加入; v0.2 接入微信支付(BP解锁/付费加入/发布费/分佣); v0.3 合伙人。

### 7.1 BP会员(按项目) **[v0.2]**
- 按项目独立收费解锁商业计划书
- 解锁后永久可读该项目BP

### 7.2 加入项目
- B端设置: 免费 或 收费(1-999元)
- 收费上限由总后台设定
- 付费加入后解锁项目方联系信息
- **v0.1**: 仅免费加入模式
- **v0.2**: 收费加入 + 微信支付

### 7.3 申请合伙人 **[v0.3]**
- 支付费用成为城市运营中心合伙人

### 7.4 发布项目
- 新项目发布支付费用 **[v0.2 接入支付; v0.1 人工审核免费发布]**

## 8. 分佣体系(一级)

### 8.1 分佣规则
- **仅一级推荐**: A推荐B,B加入项目付费/B解锁BP付费 → A获分佣
- 推荐关系: 用户分享二维码/链接,新用户注册即绑定上级

### 8.2 分佣来源
- C端推荐新用户加入项目付费
- C端推荐新用户解锁BP付费

### 8.3 分佣比例
- 由B端或平台配置(待定具体比例)

## 9. 资源互换

### 9.1 用户资源信息
- 每用户可填: 拥有资源、需求资源、收藏项目
- 全平台用户列表可见(资源/需求/收藏)

**隐私边界**:
| 项 | 规则 |
|---|---|
| 公开字段白名单 | 昵称、资源/需求描述、所属B端、收藏项目(公开);**不含**手机/微信/身份证 |
| 联系方式脱敏 | 手机/微信默认隐藏,仅互换授权后可见(见9.2) |
| 可见性设置 | 用户可设置资源/资料 `公开`/`仅B端内`/`隐藏` |
| 搜索范围 | 按地区/行业/资源关键词搜索;不暴露精确位置 |
| 举报 | 用户可举报他人资料/资源;平台审核 |
| 拉黑 | 用户可拉黑他人,被拉黑方不可见其资料/不可发起互换 |
| 封禁 | 平台可封禁违规用户,封禁后资料下线、互换禁用 |

### 9.2 微信互换
- 默认微信不可见
- 申请互换 → 对方同意 → 双方互见微信
- 互换设置: 默认同意 或 审核同意(用户配置)

## 10. 非功能需求

| 项 | 要求 |
|---|---|
| 多租户隔离 | B端数据双层隔离(应用层ORM + PostgreSQL RLS),互不可见 |
| 租户信任 | 租户上下文由JWT/服务端解析,不信任客户端 X-B-ID 头 |
| 支付 | 微信支付,平台收口分账;含退款冲正 |
| 分佣闭环 | 订单状态机 + 佣金状态机(待结算/冻结/已结算/冲正) |
| 性能 | 小程序首屏 < 2s,列表分页 |
| 安全 | 企查查链接防注入,BP付费校验,B端首登短信链接设密 |
| 实名 | 分佣提现需实名+身份证 |
| 审核 | 项目发布/用户注册可配置自动或人工 |
